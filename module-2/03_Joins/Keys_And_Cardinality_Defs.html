Keys are used to create relationships between tables.

Natural keys are formed from values in the real world (SSN, ISBN, tax id)

Surrogate keys are artificially created by the application and identify a unique record.

Primary keys uniquely identify each row within a table. They cannot be duplicated within that table and cannot be null. It is typically a single column but can also be comprised of multiple columns.

Foreign keys exist in other tables and are used to reference a primary key in the source table.

Cardinality refers to the maximum number of times that an instance in one entity can be associated with instances in a related entity, along with the minimum number of times it must be associated.

SQL Joins allow us to create queries that produce data from one or more tables

Joins can be inner or outer.
